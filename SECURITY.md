# Security policy

## Supported versions

The default branch and the latest published release receive security review.
Versioned data artifacts remain immutable; a security-sensitive correction is
published as a new release with new hashes.

## Report a vulnerability privately

Do not open a public issue for credentials, private data exposure, repository
write access, workflow injection or another exploitable condition. Use the
repository's **Security > Report a vulnerability** form, or email
<support@faceswapai.com> with the subject `Open research security report`.

Include the affected URL or file, reproduction steps, expected impact and any
temporary mitigation. Do not include customer faces, account credentials or
other private user data unless FaceSwapAI explicitly provides a protected
transfer method.

Data corrections, source updates and methodological questions are not security
reports; use the public data-correction issue template for those.
