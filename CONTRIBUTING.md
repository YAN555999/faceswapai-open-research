# Contributing corrections

This repository accepts factual corrections, reproducibility reports and
clarifications to published limitations. It is not a general product-support
queue.

## Report a data issue

Open a data-correction issue and include:

1. The exact file, field or canonical page affected.
2. The current value and proposed value.
3. A first-party source URL or reproducible observation supporting the change.
4. Whether the issue changes a conclusion, method, source boundary or only presentation.

Do not attach private faces, customer uploads, identity documents, account
data, credentials or unpublished personal information. Product and billing
support belongs at <mailto:support@faceswapai.com>.

## Review and versioning

FaceSwapAI Editorial reviews the evidence against the canonical method and
source boundary. Accepted changes are recorded publicly. A correction that
changes a released artifact receives a new version, manifest, SHA-256 set and
release notes; an existing release tag is never silently rewritten.

Vendor-claim observations must cite the vendor's own page and observation date.
They must not be rewritten as independent product tests. Measurement-study
changes must preserve the inputs, transformations, detector or score version,
variables and limitations needed to interpret the result.
